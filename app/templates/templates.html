How Word document tags work

Open your .docx in Word and write a tag wherever the value should appear. Tags use double curly braces:

  • {% raw %}{{ part_number }}{% endraw %} — replaced with the value of the field whose key is part_number.
  • {% raw %}{{ project_name }}{% endraw %}, {% raw %}{{ project_start_date }}{% endraw %}, {% raw %}{{ project_health }}{% endraw %}, {% raw %}{{ today }}{% endraw %} — always available.
  • {% raw %}{% for m in milestones %}{{ m.title }} – {{ m.target_date }}{% endfor %}{% endraw %} — loop over project data.

Each tag must match the key of a field below. Upload your .docx, link the tags to fields, then click Download filled for a project.

Field syntax & examples

One line per field, with six pipe-separated parts. Empty parts are allowed (just leave them blank between two |).

key | label | type | aliases | required/optional | value
  • type = string — plain text. Value is anything: ABC-12345, Acme Corp, etc.
  • type = excel_cell — a single cell reference of the form FileName.xlsx!Sheet1!B12 (or just FileName.xlsx!B12 if there is one sheet). Example value: Quote.xlsx!B12.
  • type = excel_table — a rectangular range of the form FileName.xlsx!A1:H40. Example value: BOM.xlsx!A1:H40.

aliases lists alternate column names other documents use for the same field, separated by commas (e.g. PN,Item Number). Used by the field-homologation panel.

required means the field counts toward completion. Use optional for ignore-if-blank.
